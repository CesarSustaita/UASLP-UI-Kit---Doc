La paleta de colores del **UASLP UI Kit** es un pilar fundamental para asegurar la **consistencia visual y la accesibilidad** en todas las interfaces de la Universidad Autónoma de San Luis Potosí (UASLP), tanto web como móvil. Nuestro objetivo es acelerar la creación de nuevas interfaces, manteniendo siempre la identidad de la institución.

---

## Filosofía del Color

Nuestra filosofía de color se centra en **mantener y realzar los colores vigentes de la UASLP**, creando una experiencia de usuario que refleje la seriedad, la confianza y la tradición académica. Hemos desarrollado una paleta basada en escalas de azules y grises, que son los colores distintivos de la universidad. Estos colores base se complementan con tonos funcionales (verde, rojo, amarillo) para asegurar una comunicación clara y una interacción intuitiva en la interfaz.

El **alcance** principal de esta paleta es para todas las **interfaces digitales de la universidad**, incluyendo aplicaciones web y móviles.

## Guías de Uso y Aplicación

Para asegurar la coherencia y accesibilidad en el uso de los colores del UASLP UI Kit, ten en cuenta las siguientes directrices:

### Accesibilidad

La accesibilidad es primordial. Todos los colores de texto y elementos interactivos deben cumplir con las directrices **WCAG 2.1 AA** para garantizar un contraste adecuado. Esto asegura que nuestras interfaces sean utilizables por personas con diversas capacidades visuales, incluyendo aquellos con daltonismo.

* **Contraste mínimo:** Asegúrate de que la relación de contraste sea de al menos **4.5:1** para texto normal y **3:1** para texto grande (18pt o 14pt negrita). Puedes usar herramientas como [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) para verificar el contraste.
* **Percepción del Color:** Evita transmitir información únicamente a través del color. Siempre acompaña las señales de color (ej. rojo para error) con iconos, texto o patrones para que la información sea accesible para usuarios con daltonismo.

### Jerarquía Visual

Los colores deben usarse estratégicamente para establecer una clara jerarquía visual:

* **Colores primarios (`blue-dark-500` - `blue-dark-700`):** Para elementos clave de la marca, botones de acción principal y encabezados que requieren máxima atención.
* **Colores secundarios (`blue-light-500` - `blue-light-700`):** Para acciones secundarias, elementos interactivos o información destacada que complementa el contenido principal.
* **Grises (`gray` y `gray-blue`):** Utilizados para texto general, fondos, bordes y elementos de interfaz que necesitan ser neutrales para no distraer. Los tonos más oscuros para texto y los más claros para fondos y separadores.
* **Colores funcionales (`green`, `red`, `yellow`):** Exclusivamente para comunicar estados de éxito, error o advertencia, respectivamente. Su uso debe ser intencional y limitado.

### Uso Adecuado y Errores Comunes

* **Do's (Qué hacer):**
    * Utiliza los **`blue-dark-500`** y **`blue-light-500`** como colores principales de la marca y para enlaces activos.
    * Aplica **`green-500`** para confirmaciones de éxito y retroalimentación positiva.
    * Usa **`red-500`** para indicar errores críticos o acciones que requieren precaución inmediata.
    * Aplica **`yellow-500`** para advertencias o información que necesita atención pero no es crítica.
    * Emplea la escala de **`grey`** para texto secundario, fondos neutros, bordes y separadores, asegurando siempre el contraste.
    * Asegúrate de que los colores de interacción (hover, active, focus) sean variaciones sutiles de los colores base para mantener la coherencia.
* **Don'ts (Qué no hacer):**
    * No uses colores funcionales (verde, rojo, amarillo) para decoración o sin un propósito semántico claro.
    * Evita mezclar tonos de azul de la paleta **`BLUE-DARK`** y **`BLUE-LIGHT`** indiscriminadamente, a menos que sea para un propósito específico y documentado.
    * No cambies los valores Hex o RGB de los tokens definidos. Si necesitas un nuevo tono, documentarlo y agregarlo a la paleta.
    * No utilices colores de texto que no cumplan con los estándares de contraste de accesibilidad.
